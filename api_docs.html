📖 SlideCraft API 文档

SlideCraft 使用两步式API设计：先生成内容，再生成PPTX文件。

🔄 API 调用流程

完整流程：
1️⃣ 调用 POST /api/generate 生成PPT内容（AI生成大纲）
2️⃣ 调用 POST /api/download_pptx 下载PPTX文件（根据内容生成）

📝 API 端点详解

POST /api/generate

功能：生成PPT内容大纲（AI生成）

请求体（JSON）：

{
  "topic": "PPT主题（必需）",
  "slides": 8,
  "style": "简洁明了",
  "model": "minimax-m27"
}

响应（JSON）：

{
  "content": "生成的PPT内容大纲（Markdown格式）"
}

POST /api/download_pptx

功能：下载PPTX文件

请求体（JSON）：

{
  "topic": "PPT主题（可选，用于文件名）",
  "content": "第一步生成的content内容（必需）",
  "theme": "森林墨"
}

响应：PPTX文件（二进制流）

响应头：

Content-Type: application/vnd.openxmlformats-officedocument.presentationml.presentation
Content-Disposition: attachment; filename*=UTF-8''主题.pptx

POST /api/generate_ppt

功能：生成HTML PPT预览（可选）

请求体（JSON）：

{
  "topic": "PPT主题（可选）",
  "content": "第一步生成的content内容（必需）",
  "theme": "森林墨"
}

响应（JSON）：

{
  "success": true,
  "html": "..."
}

💻 Python 调用示例

import requests
import json

# 第一步：生成PPT内容
print("=== 第一步：生成PPT内容 ===")
generate_data = {
    "topic": "人工智能应用",
    "slides": 8,
    "style": "专业严谨",
    "model": "minimax-m27"
}

response = requests.post("http://localhost:5000/api/generate", json=generate_data)
print(f"Status: {response.status_code}")

if response.status_code != 200:
    print(f"Error: {response.text}")
    exit(1)

result = response.json()
content = result.get("content", "")
print(f"Content length: {len(content)}")

# 第二步：下载PPTX文件
print("\n=== 第二步：下载PPTX文件 ===")
download_data = {
    "topic": "人工智能应用",
    "content": content,
    "theme": "森林墨"
}

response = requests.post("http://localhost:5000/api/download_pptx", json=download_data)

print(f"Status: {response.status_code}")
print(f"Content-Type: {response.headers.get('Content-Type')}")

if response.status_code == 200:
    with open("output.pptx", "wb") as f:
        f.write(response.content)
    print(f"PPTX文件已保存: output.pptx ({len(response.content)} bytes)")
else:
    print(f"Error: {response.text}")

⚠️ 常见错误

400 错误："请输入 PPT 主题"
原因：缺少 topic 参数
解决：在请求体中添加 "topic": "你的PPT主题"
400 错误："请先生成 PPT 大纲"
原因：调用 /api/download_pptx 时缺少 content 参数
解决：先调用 /api/generate 获取 content，再传递给 /api/download_pptx
500 错误："请求超时，请稍后重试"
原因：AI模型API调用超时
解决：检查API Key和网络连接，或增加超时时间

📚 其他API端点

GET /api/themes

功能：获取可用的PPT主题列表

GET /health

功能：健康检查

💡 为什么使用两步式设计？

  1. 灵活性：用户可以先预览生成的内容，确认无误后再生成PPTX
  2. 性能：HTML预览可以快速展示，PPTX生成较慢但可以按需下载
  3. 可调试性：如果PPTX生成失败，可以保留生成的内容用于调试
